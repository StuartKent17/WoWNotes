Equipment Manager Update
After the hunter ammunition changes, it's time for the Equipment Manager to be pulled out of the patch because of bugs. We can assume that it will be deployed in a minor patch after the release of 3.1 on live servers. (Again, nothing official about this)

2009 Arena Tournament Phase 3 Ending
Phase 3 of the 2009 Arena Tournament will be ending next Monday, April 6, at 9:00 p.m. PDT. Please keep in mind that you have until the end of this phase to complete the 200 rated 3v3 games required to qualify for prizes. Participants must also remain on their team for the duration of the tournament to ensure prize eligibility.

The Arena Tournament realms will be shut down for a week in preparation for Phase 4 of the tournament (the invitational ladder), which will determine who advances to the regional finals. Phase 4 will begin on Tuesday, April 14, after maintenance. For more information on the tournament, please visit our Arena Tournament information page at http://www.worldofwarcraft.com/pvp/tournament/index.xml (http://www.worldofwarcraft.com/pvp/tournament/index.xml) ; to discuss the tournament with your fellow players, please visit the Arena Tournament Forum. Thank you to all of the participants, and good luck to everyone who qualified for the invitational ladder!

There are like 3+ equipment add ons that have been consistently updated and functional after every patch. Probably written by kids sitting in a basement. I'm just curious, if they can do it...why cant Blizzard?


Blizzard typically adds new APIs to the game client itself to facilitate features like this. Add-ons need to sort of hack around limitations in the client. True, they get it accomplished, but not elegantly. Blizzard is heavily modifying the APIs around item management in order to cleanly support this new management system. These changes greatly benefit the add-on authors as well, as they can refactor their code to make use of the improvements.

However, any API changes to the client must be highly tested. If those APIs also require server-side changes, the potential issues are magnified further. From a software lifecycle standpoint, this new feature represents a great deal of effort on their part.

The independent add-ons will still be updated and will work while we wait for this feature to be released. In the meantime, just be aware that adding a new built-in feature to the game is NOT the same thing as just writing an add-on.

There are like 3+ equipment add ons that have been consistently updated and functional after every patch. Probably written by kids sitting in a basement. I'm just curious, if they can do it...why cant Blizzard?

And all three of those addons have been around for more then a year receiving regular updates. You expect Blizzard to duplicate all that in just a few months?

Plus, all of those addons are Lua only implementations storing their information in the saved variables. Blizzard's system has to store it server side, which means development within the game's own Lua UI, changes to the actual client software, and changes to the server software & Database. If an addon has a bug, it's a simple procedure to release a fix. If something in WoW has a bug, they have to go through a complicated patching procedure that can take weeks.

This is why Blizzard doesn't make a lot of changes to ingame UI, it requires a LOT more work compared to what the addon devs do.
