Patch 4.3.2 on Test Realms
A new Patch 4.3.2 is being deployed on test realms as we speak, this is a "minor bug fix patch." according to the official notes but we do not have more details for the moment.

World of Warcraft Optimization
The launcher now optimizes your game folder and tries to improve game performance/reduce the size of the install on your hard drive.

I made this (http://vps.marlamin.nl/u/Screenshot-2012-01-03_19.56.01.png) picture yesterday, to explain. Basically, the tons of patch data that has been gathered up since 4.0 will be added into the base MPQs, saving some space and speed. I imagine this happening every few major patches.

To explain a little more:

The MPQ is Blizzard's data storage file - everything from art to spells. Before Cataclysm, each patch would over-write the original MPQ data, leaving just "base" files.

Since Cataclysm patches have become separate MPQ files, each containing a set of changes since the previous patch. So each time the game needs to load data, it not only has to check the base file, but read through all the patches in sequence, to check for changes. The Cataclysm-style allowed data to be streamed, and small fixes to be distributed easily, but does have the disadvantage that after a while (especially on PTRs), there are tens of patches to be checked, which presumably slows down load times a little.

What the optimization appears to do is combine previous patch MPQs into base MPQs.

Now, I presume that streaming still works like before: The optimization is simply bringing the client's files in line with the latest base file - the base file that one would get streamed if one needed it. That would explain why the optimization cannot be skipped - because otherwise there would not be a source of streamed data, if your client needed it. It also suggests that optimization will not be a common process - likely only around the time of major patches. And this reduction of patch files into base files was something that the game client has done before during major patches - it just didn't announce it as a special process.

What would the advantages be of running WoW with a 64 bit client?

There wouldn't be many benefits that you would immediately notice. The ability for the game to utilize more than 4 GB of memory is the main point of interest, and even then that's something that wouldn't affect players for several more years, if it ever were to at all. 64-bit is a great way of making the game more "future proof", so to speak. The amount of average memory in a computer since WoW's release in 2004 has probably more than quadrupled. If that trend continues along with WoW's continued growth and evolution, there's no doubt that we could be exhausting the 32-bit address space limit altogether years down the road.
